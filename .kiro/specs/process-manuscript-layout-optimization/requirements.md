# 需求文档

## 简介

本文档规定了 SUSY 稿件管理系统中 Process Manuscript 页面布局和内容组织优化的需求。优化重点是通过调整左右列比例和重新组织卡片内容来改善视觉层次和信息架构，以更好地区分稿件相关信息和系统/流程相关信息。

## 术语表

- **Process_Manuscript_Page（稿件处理页面）**: 显示 SUSY 系统中正在处理的稿件详细信息的页面组件
- **Layout_System（布局系统）**: 基于 CSS Grid 的布局系统，控制列比例和响应式行为
- **Left_Column（左列）**: 两列布局的左侧，显示稿件本身的信息
- **Right_Column（右列）**: 两列布局的右侧，显示系统和流程信息
- **Card_Component（卡片组件）**: 将相关信息字段分组的内部区块组件（例如：稿件元数据、作者、文件）
- **Golden_Ratio（黄金比例）**: 约为 0.618:0.382 的数学比例，用于美观的视觉比例
- **Manuscript_Information（稿件信息）**: 描述稿件本身的数据（元数据、作者、文件）
- **System_Information（系统信息）**: 描述工作流程、状态和系统流程的数据（状态、角色、审稿、生产）

## 需求

### 需求 1: 调整布局比例

**用户故事：** 作为查看 Process Manuscript 页面的用户，我希望左列比右列更宽，以便稿件内容获得适当的视觉重点。

#### 验收标准

1. 布局系统应使用 0.618:0.382（黄金比例）的列比例
2. 当视口宽度低于 1400px 时，布局系统应显示全宽的单列布局
3. 当视口宽度达到或超过 1400px 时，布局系统应显示黄金比例的两列布局
4. 布局系统应保持 1.5rem 的一致列间距

### 需求 2: 重组左列内容

**用户故事：** 作为用户，我希望左列只包含稿件本身的信息，以便我可以快速找到关于稿件本身的信息。

#### 验收标准

1. 左列应包含稿件元数据卡片组件
2. 左列应包含作者卡片组件
3. 左列应包含文件卡片组件
4. 左列不应包含任何系统流程或工作流信息
5. 左列应按以下顺序显示卡片组件：稿件元数据、作者、文件

### 需求 3: 重组右列内容

**用户故事：** 作为用户，我希望右列只包含系统和流程信息，以便我可以将工作流状态和系统操作与稿件内容分开跟踪。

#### 验收标准

1. 右列应包含状态与流程卡片组件
2. 右列应包含人员与角色卡片组件
3. 右列应包含审稿与质量检查卡片组件
4. 右列应包含生产与出版卡片组件
5. 右列应包含预印本发布卡片组件
6. 右列应包含最近活动卡片组件
7. 右列应按以下顺序显示卡片组件：状态与流程、人员与角色、审稿与质量检查、生产与出版、预印本发布、最近活动

### 需求 4: 保持稿件元数据卡片结构

**用户故事：** 作为用户，我希望稿件元数据卡片包含所有稿件识别和分类字段，以便我可以了解稿件是什么。

#### 验收标准

1. 稿件元数据卡片应显示稿件 ID 字段
2. 稿件元数据卡片应显示 DOI 字段
3. 稿件元数据卡片应显示稿件状态字段
4. 稿件元数据卡片应显示标题字段
5. 稿件元数据卡片应显示标签字段
6. 稿件元数据卡片应显示文章类型字段
7. 稿件元数据卡片应显示大型综述字段
8. 稿件元数据卡片应显示注册报告阶段字段
9. 稿件元数据卡片应显示期刊字段（带内联编辑功能）
10. 稿件元数据卡片应显示栏目字段
11. 稿件元数据卡片应显示特刊字段
12. 稿件元数据卡片应显示主题字段
13. 稿件元数据卡片应显示会议名称字段
14. 稿件元数据卡片应显示文件类型字段
15. 稿件元数据卡片应显示页数字段
16. 稿件元数据卡片应显示稿件来源字段
17. 稿件元数据卡片应显示作者贡献字段
18. 稿件元数据卡片应显示稿件负责人字段

### 需求 5: 保持作者卡片结构

**用户故事：** 作为用户，我希望作者卡片包含所有作者相关信息和警告，以便我可以识别谁撰写了稿件以及任何作者相关问题。

#### 验收标准

1. 作者卡片应显示提交作者字段
2. 作者卡片应显示所有作者字段
3. 当存在被阻止员工警告时，作者卡片应显示被阻止员工警告字段（带危险样式）
4. 当存在会员不匹配时，作者卡片应显示会员不匹配字段（带警告样式）
5. 当存在作者角色时，作者卡片应显示作者角色（主编/客座编辑）字段

### 需求 6: 保持文件卡片结构

**用户故事：** 作为用户，我希望文件卡片显示所有稿件文件及下载操作，以便我可以访问稿件文档和补充材料。

#### 验收标准

1. 文件卡片应显示所有稿件文件的列表
2. 对于文件列表中的所有文件，文件卡片应显示文件图标、文件名、文件元数据和下载按钮
3. 文件卡片应显示上传稿件按钮
4. 文件卡片应显示显示日志按钮
5. 文件卡片应支持的文件类型包括：Word 文档、PDF、图像、压缩包和外部链接

### 需求 7: 保持状态与流程卡片结构

**用户故事：** 作为用户，我希望状态与流程卡片显示工作流状态和时间线信息，以便我可以跟踪稿件在系统中的进度。

#### 验收标准

1. 状态与流程卡片应显示当前状态字段（带内联编辑功能）
2. 当存在拒稿原因时，状态与流程卡片应显示拒稿原因字段
3. 当存在待定拒稿原因时，状态与流程卡片应显示待定拒稿原因字段（带警告样式）
4. 当存在撤稿原因时，状态与流程卡片应显示撤稿原因字段
5. 状态与流程卡片应显示收到提交字段（带日期和已过天数）
6. 状态与流程卡片应显示大修次数字段
7. 当存在接受日期时，状态与流程卡片应显示接受日期字段
8. 当存在发表日期时，状态与流程卡片应显示发表日期字段
9. 状态与流程卡片应显示准备发表字段（带切换开关）

### 需求 8: 保持人员与角色卡片结构

**用户故事：** 作为用户，我希望人员与角色卡片显示所有人员分配，以便我可以识别谁负责稿件处理的不同方面。

#### 验收标准

1. 人员与角色卡片应显示分配编辑字段（带内联编辑功能）
2. 人员与角色卡片应显示校对编辑发送校样字段
3. 人员与角色卡片应显示分配的校对编辑字段
4. 当存在分配校对编辑日期时，人员与角色卡片应显示分配校对编辑日期字段
5. 人员与角色卡片应显示分配的英文编辑字段
6. 人员与角色卡片应显示关注者字段
7. 当存在主题监督者时，人员与角色卡片应显示主题监督者字段
8. 当存在客座编辑时，人员与角色卡片应显示客座编辑（特刊/主题）字段

### 需求 9: 保持审稿与质量检查卡片结构

**用户故事：** 作为用户，我希望审稿与质量检查卡片显示所有审稿、质量控制和合规信息，以便我可以评估稿件的审稿状态并识别任何问题。

#### 验收标准

1. 审稿与质量检查卡片应显示开放同行评审字段
2. 审稿与质量检查卡片应显示招募审稿人字段
3. 审稿与质量检查卡片应显示反驳上传字段
4. 审稿与质量检查卡片应显示提交天数（副主编 SLA）字段
5. 审稿与质量检查卡片应显示质量检查通过字段（带成功或危险样式）
6. 当存在质量检查评论时，审稿与质量检查卡片应显示质量检查评论字段
7. 审稿与质量检查卡片应显示资助信息字段
8. 审稿与质量检查卡片应显示数据可用性声明字段
9. 当存在 CCDC 存储编号时，审稿与质量检查卡片应显示 CCDC 存储编号字段
10. 审稿与质量检查卡片应显示利益冲突字段
11. 审稿与质量检查卡片应显示已在其他地方发表字段
12. 审稿与质量检查卡片应显示伦理问题字段（带适当样式）
13. 审稿与质量检查卡片应显示有害主题字段（带适当样式）
14. 审稿与质量检查卡片应显示争议主题字段（带适当样式）
15. 审稿与质量检查卡片应显示国家观察名单字段（带适当样式）
16. 审稿与质量检查卡片应显示公开可用数据集字段
17. 审稿与质量检查卡片应显示扭曲短语检测字段（带适当样式）
18. 审稿与质量检查卡片应显示附加稿件属性字段
19. 审稿与质量检查卡片应显示同行评审前检查（白名单）字段（带适当样式）

### 需求 10: 保持生产与出版卡片结构

**用户故事：** 作为用户，我希望生产与出版卡片显示生产工作流状态，以便我可以跟踪稿件在生产阶段的进度。

#### 验收标准

1. 生产与出版卡片应显示 Proofig（首次）字段
2. 生产与出版卡片应显示 Proofig 修订后字段
3. 生产与出版卡片应显示布局检查（Merrin）字段
4. 生产与出版卡片应显示接受证书字段
5. 当存在接受日期时，生产与出版卡片应显示接受证书的生成按钮
6. 生产与出版卡片应显示英文服务状态（作者服务）字段
7. 生产与出版卡片应显示英文预编辑状态（正常）字段

### 需求 11: 保持预印本发布卡片结构

**用户故事：** 作为用户，我希望预印本发布卡片显示预印本状态信息，以便我可以跟踪预印本发布活动。

#### 验收标准

1. 预印本发布卡片应显示预印本状态字段
2. 当已发送预印本邮件时，预印本发布卡片应显示预印本邮件已发送字段
3. 当预印本未上传时，预印本发布卡片应显示预印本未上传字段（带警告样式）
4. 当存在预印本 CTA 上传时，预印本发布卡片应显示预印本 CTA 上传字段

### 需求 12: 保持最近活动卡片结构

**用户故事：** 作为用户，我希望最近活动卡片显示稿件事件的时间顺序历史，以便我可以了解对稿件采取了哪些操作。

#### 验收标准

1. 最近活动卡片应显示历史项目列表
2. 对于所有历史项目，最近活动卡片应显示日期和事件描述
3. 最近活动卡片应按时间顺序显示历史项目
4. 最近活动卡片应使用视觉分隔符分隔历史项目

### 需求 13: 保持响应式行为

**用户故事：** 作为在不同屏幕尺寸上的用户，我希望布局能够适当调整，以便我可以在任何设备上查看稿件信息。

#### 验收标准

1. 当视口宽度低于 1400px 时，布局系统应将所有卡片堆叠在单列中
2. 当视口宽度达到或超过 1400px 时，布局系统应显示黄金比例的两列布局
3. 布局系统应在所有断点保持一致的间距和内边距
4. 布局系统应在所有断点保留所有卡片内容和功能

### 需求 14: 保持视觉样式

**用户故事：** 作为用户，我希望视觉外观与现有设计系统保持一致，以便页面与应用程序的其余部分感觉协调。

#### 验收标准

1. 稿件处理页面应保持文本、背景和边框的现有配色方案
2. 稿件处理页面应保持标题、标签和正文文本的现有排版样式
3. 稿件处理页面应保持卡片内的现有间距和内边距值
4. 稿件处理页面应保持卡片的现有边框半径和阴影样式
5. 稿件处理页面应保持状态指示器和标签的现有徽章和标签样式

### 需求 15: 保持交互功能

**用户故事：** 作为用户，我希望所有交互元素继续像以前一样工作，以便我可以编辑字段和执行操作而不会中断。

#### 验收标准

1. 稿件处理页面应保留期刊字段的内联编辑功能
2. 稿件处理页面应保留当前状态字段的内联编辑功能
3. 稿件处理页面应保留分配编辑字段的内联编辑功能
4. 稿件处理页面应保留准备发表字段的切换开关功能
5. 稿件处理页面应保留所有文件项的下载按钮功能
6. 稿件处理页面应保留上传稿件按钮功能
7. 稿件处理页面应保留显示日志按钮功能
8. 稿件处理页面应保留接受证书的生成按钮功能

# Summary 接口场景说明

## 📋 场景总览

根据 document_hash 的尾数（最后一位数字），返回不同的问题场景。

| 尾数 | 场景描述 | HTTP 状态码 |
|------|---------|------------|
| 0 | 返回空 result | 200 |
| 1 | authors 只有普通问题 + AI 写作有问题 | 200 |
| 2 | authors 有严重问题 | 200 |
| 3 | references 只有普通问题 | 200 |
| 4 | references 有严重问题 | 200 |
| 5 | authors + references 都只有普通问题 | 200 |
| 6 | authors + references 都有严重问题 + peer_review 有问题 | 200 |
| 7 | 所有项都有问题 | 200 |
| 8 | 无问题 | 200 |
| 9 | 返回错误 | 500 |

---

## 🔍 详细场景说明

### 场景 0：空 result
```json
{
  "date": "2026-05-14T...",
  "note": null,
  "result": {}
}
```

---

### 场景 1：authors 普通问题 + AI 写作问题

**特征：**
- ✅ `authors.warning_count` > 0（1-5 随机）
- ✅ `authors.severe_warning_count` = 0
- ✅ `ai_writing.proportions.human_proportion` < 0.7
- ✅ `ai_writing.proportions.maybe_ai_proportion` > 0
- ✅ `ai_writing.proportions.likely_ai_proportion` > 0
- ✅ `references` 无问题
- ✅ `peer_review_reports` 无问题

**示例响应：**
```json
{
  "result": {
    "issues_summary": {
      "authors": {
        "warning_count": 3,
        "severe_warning_count": 0
      },
      "references": {
        "warning_count": 0,
        "severe_warning_count": 0
      },
      "ai_writing": {
        "severe_warning_count": 2,
        "proportions": {
          "human_proportion": 0.55,
          "maybe_ai_proportion": 0.25,
          "likely_ai_proportion": 0.20
        }
      },
      "peer_review_reports": [
        {
          "human_proportion": 1.0,
          "maybe_ai_proportion": 0.0,
          "likely_ai_proportion": 0.0
        }
      ]
    }
  }
}
```

## 💡 关键特性

### 普通问题 vs 严重问题

- **只有普通问题**：`warning_count` > 0，`severe_warning_count` = 0
- **只有严重问题**：`warning_count` = 0，`severe_warning_count` > 0
- **两者都有**：`warning_count` > 0，`severe_warning_count` > 0（仅场景 7）

### AI 写作检测

- **有问题**：`human_proportion` < 0.7
- **无问题**：`human_proportion` = 1.0
- **场景 1 特殊要求**：`maybe_ai_proportion` 和 `likely_ai_proportion` 都必须 > 0

### 比例总和

所有 AI 相关的比例总和必须等于 1.0：
```
human_proportion + maybe_ai_proportion + likely_ai_proportion = 1.0
```

## 📝 注意事项

1. 所有场景的 `report_link` 固定为 `https://deployment.susy.mdpi.dev/`
2. 所有场景的 `note` 固定为 `null`
3. 警告数量是随机生成的，每次请求可能不同
4. AI 比例是随机生成的，但保证总和为 1.0
5. 场景 1 确保 `maybe_ai_proportion` 和 `likely_ai_proportion` 都不为 0
